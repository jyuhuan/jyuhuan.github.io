Building Styled Text with NeonSign

Learn to use NeonSign’s declarative syntax to build styled text.

Overview

Use this step-by-step tutorial to learn to use NeonSign’s syntax from simple examples. To browse examples that fully demonstrate NeonSign’s capabilities, see Example Gallery.

Topics covered in this tutorial:

  • How to style Python strings using the s() function.

  • How to nest s() expressions to create complex designs.

Syntax at a Glance

Wrap your contents with the s() function, and call one or more styling methods to describe the styles you want for your content.

You can also mix plain strings and other s() expressions, separated by commas, as the contents of your styled string:


Browse all styling methods available in the documentation of the StyledString class.

Start with s()

Let’s add some styles to a simple Python string, 'example'. First, wrap it with the s() function:

from neonsign import s

s('example')

This creates a starting point for styles to be added.

Note

What does the s() function do? It creates an instance of StyledString, which will have your string as its content. In the code above, the expression s('example') creates a StyledString with 'example' as its content.

To render a styled string to your terminal, simply pass your s() expression to Python’s built-in print() function:

from neonsign import s

print(s('example'))

Here’s what a terminal might print:


The styled string created so far doesn’t have any styles yet. So when it’s printed, its appearance will be the same as it usually appears.

Order of styling methods matters

Note that if you change the order in a chain of styling method calls, you may get different results.

Take padded() and background() as an example. If padded() is used before background(),

from neonsign import s, Color

s('example').padded().background(Color.GREEN)

the background color will be applied to the padded spaces:

However, if background() is used before padded(),

from neonsign import s, Color

s('example').background(Color.GREEN).padded()

the background color will only be applied to the original string. The padded spaces won’t have the background color specified:

Choose the order of the styling methods that best fits your intended design. When in doubt, print it to the terminal to test the rendering result.

Nesting s() expressions

So far, the s() function was used only to create a styled string from a simple Python string. But s() can actually take more than one argument. Each argument can be either a Python string, or another StyledString. This allows s() expressions to be nested, to create complex designs out of simpler pieces. For example:

from neonsign import s, Color

s(
    s('Hello').background(Color.YELLOW),
    s('World').italic().underlined(),
    '!'
).bold().background(Color.GREEN)

Renders as:


Here, the resulting styled string consists of

  • the word “Hello” in bold, with a yellow background, followed by

  • the word “World” in bold, underlined and italic, with a green background, followed by

  • the exclamation mark (!) in bold, with a green background.

Important

When you have nested s() expressions, the styling methods on an outer expression applies to inner expressions as well. This is useful for applying the same style to a group of strings. However, the styling methods on an inner expression can override the outer styles.

In the example above, the outer expression has two styling methods, .bold() and .background(Color.GREEN). This tells NeonSign to render all contents in bold and with a green background. Indeed, the entire string “HelloWorld!” is rendered in bold. However, because the inner expression s('Hello') has its own background styling method, .background(Color.YELLOW), the green background specified in the outer expression was overridden to yellow, just for “Hello”.
